# Implement Diyou PWA Portal Spec

## Why
The project foundation and specifications are established. We now need to execute the actual implementation of the Diyou internal website, encompassing the backend REST API, database operations, media storage, and the Vue 3 frontend according to the Claude-style design system.

## What Changes
- Implement backend authentication and user management
- Implement backend APIs for members, families, media, works, matches, and annual events
- Implement backend integration with Qwen for the Annual Planner Skill using RAG
- Implement frontend UI components based on the Claude design system (`DESIGN.md` and `docs/design-system.md`)
- Implement frontend pages (Home, History, Media, Works, People, Upload, Planner)
- Connect frontend with backend APIs
- Ensure PWA offline capabilities are functional

## Impact
- Affected specs: `docs/api.md`, `docs/database.md`, `docs/design-system.md`, `docs/information-architecture.md`, `docs/pwa.md`, `docs/storage.md`, `docs/skill-annual-planner.md`
- Affected code: `/workspace/backend/*`, `/workspace/frontend/*`

## ADDED Requirements
### Requirement: Backend API Implementation
The system SHALL provide all REST APIs defined in `docs/api.md` using Fastify and Prisma.

#### Scenario: Success case
- **WHEN** a client sends a valid request to `/api/v1/members`
- **THEN** the backend responds with a 200 OK and the list of members

### Requirement: Frontend UI Implementation
The frontend SHALL implement all pages and components according to the Claude-style design system and the information architecture.

#### Scenario: Success case
- **WHEN** a user navigates to `/history`
- **THEN** the timeline is rendered correctly with appropriate styling and interactivity

### Requirement: Media Storage
The system SHALL store uploaded media files on the local disk as specified in `docs/storage.md`.

#### Scenario: Success case
- **WHEN** an authorized user uploads a photo
- **THEN** the file is saved to the local disk and its metadata is stored in the database

### Requirement: Qwen Annual Planner Integration
The system SHALL provide a Planner Skill that uses Qwen to generate an annual event plan based on user constraints and knowledge base documents.

#### Scenario: Success case
- **WHEN** a user submits constraints for an annual event
- **THEN** the system returns a structured plan generated by Qwen, including citations from the knowledge base
