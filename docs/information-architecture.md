# 信息架构与路由规格（移动优先）

## 目标
- 主要在手机端使用，所有核心操作可单手完成
- 全站“无数据”时保持可用：每个页面都有空状态引导
- 前后端分离：前端只负责 UI/交互；数据通过 API 拉取

## 全局导航
- 顶部：站点名称 + 多语言切换
- 二级：横向可滚动主菜单（移动端友好）
- 权限：未登录只允许访问登录页与公开页（若定义为公开）；需要权限的入口在无权限时隐藏或禁用

## 路由表
| 页面 | 路由 | 说明 |
|---|---|---|
| 主页 | / | 球队简介、入口菜单、多语言切换 |
| 历史页 | /history | 时间轴（2015–2026 + 更多），进入年度页 |
| 年度页 | /history/:year | 单年度聚合：媒体、作品、比赛、聚会 |
| 照片/视频 | /media | iPhone 照片库逻辑的移动端展示（第一阶段不做人脸识别，使用手动人物标签） |
| 文集/诗集 | /works | 文章/诗词切换与搜索，支持按作者筛选 |
| 个人页（模板） | /people | 顶部三下拉：XX 号家庭、红队、蓝队；未选择时不展示明细 |
| 单人页 | /people/:id | 单人/家庭详情：关联媒体、作品、比赛参与信息 |
| 上传资料 | /upload | 创建/维护档案，上传媒体、录入作品与比赛 |
| 年会策划 | /planner | 本地知识库 + Qwen Skill 的年会方案生成界面 |

## 关键交互（摘要）
### 历史页时间轴
- 移动端采用纵向滚动为主
- 时间轴表现形式：波浪线/折线 + 年份节点（12 个圆点对应 2015–2026）
- 点击年份节点进入年度页

### 照片/视频页（第一阶段）
- 按时间分组（年/月/日的分段列表）
- 支持筛选：类型（照片/视频）、人物标签、年份
- 人物聚合采用手动标注：在上传或编辑媒体信息时选择人物标签

### 文集/诗集页
- 顶部切换：文章 / 诗词
- 搜索：作者姓名、标题关键词
- 列表 -> 详情（第二阶段可扩展富文本/Markdown）

### 个人页（模板）
- 三个下拉菜单：
  - 家庭：按“XX 号家庭”维度
  - 红队：按红队成员
  - 蓝队：按蓝队成员
- 未选择时显示空白模板与操作引导

## 空状态与错误处理
- 空状态：无数据时展示“暂无内容”与下一步建议（例如：引导到上传资料页）
- 无权限：展示明确的权限提示与登录入口
- 网络/服务错误：展示重试按钮与错误摘要（不暴露敏感信息）
