# 移动端兼容性测试矩阵

> 配合 [mobile-compatibility-plan.md](./mobile-compatibility-plan.md) 使用，用于验证 iPhone / Android / HarmonyOS 三端在浏览器态与 PWA 安装态下的核心能力。

## 1. 测试分层

| 层级 | 占比 | 工具/方式 | 职责 |
|-----|-----|----------|------|
| 静态能力检查 | 20% | 代码审查 + 单元测试 | 校验平台检测函数、策略函数输出是否符合预期。 |
| 自动化回归 | 30% | Playwright / Vitest | 浏览器级登录、导航、上传入口可见性、基础交互。 |
| 真机手工验收 | 50% | 物理设备 | 系统文件选择器、PWA 安装、后台恢复、弱网等必须真机验证的场景。 |

## 2. 静态能力检查

### 2.1 检测函数输出

| 检测函数 | 测试场景 | 期望 |
|---------|---------|------|
| `isIOS` | iPhone Safari UA | `true` |
| `isIOS` | Android Chrome UA | `false` |
| `isAndroid` | Android Chrome UA | `true` |
| `isHarmonyOS` | HarmonyOS 系统浏览器 UA | `true` |
| `isHarmonyOS` | Android UA（无 HarmonyOS 关键字） | `false` |
| `isStandalonePWA` | `display-mode: standalone` | `true` |
| `isStandalonePWA` | 普通浏览器标签 | `false` |
| `isTouchDevice` | `'ontouchstart' in window` | 按环境返回 |
| `isLikelyWebView` | 微信/企业微信等 WebView UA | `true` |

### 2.2 策略函数输出

| 策略函数 | 输入 | 期望 |
|---------|------|------|
| `shouldUseExplicitMimeList('media')` | `isHarmonyOS === true` | `true` |
| `shouldEnableDragDrop()` | `isTouchDevice === true` | `false` |
| `shouldCheckSessionOnResume()` | `isStandalonePWA === true` | `true` |
| `getAcceptForScenario('avatar')` | 任意平台 | 返回 `image/png, image/jpeg, image/webp` |

## 3. 自动化回归

### 3.1 覆盖范围

> 仅验证浏览器级行为，不强依赖自动化完成系统文件选择器验证。

| 用例 ID | 用例 | 工具 | 关键断言 |
|--------|------|------|---------|
| AUTO-01 | 未登录访问私有页面被拦截 | Playwright | 跳转 `/login` |
| AUTO-02 | 邮箱验证码登录成功 | Playwright | 跳转到首页，存在用户头像/菜单 |
| AUTO-03 | 登录态下导航到上传页 | Playwright | 页面渲染上传入口 |
| AUTO-04 | 上传页存在文件选择入口 | Playwright | `FileUploadZone` 组件可见 |
| AUTO-05 | 平台检测函数通过 Vitest | Vitest | 覆盖率 ≥ 80% |
| AUTO-06 | 文件策略函数通过 Vitest | Vitest | 覆盖率 ≥ 80% |

### 3.2 不纳入自动化的场景

以下场景因依赖真实系统文件选择器、相机、PWA 安装流程，**必须真机手测**：

- 从相册选择照片/视频
- 直接调用相机拍照
- PWA 安装与启动
- 后台恢复后的会话状态
- 弱网/离线场景

## 5. 用例详细步骤

### M-01 登录成功

1. 打开应用。
2. 输入有效邮箱与密码 / 验证码。
3. 点击登录。
4. 期望：成功进入首页，导航显示用户相关菜单。

### M-02 登录后切后台 1 分钟再恢复

1. 完成登录。
2. 切换到系统桌面或其他 App，等待 1 分钟。
3. 返回浏览器 / PWA。
4. 期望：无需重新登录，页面可操作。

### M-03 上传照片

1. 进入上传页。
2. 选择媒体上传入口。
3. 从系统相册选择 1-3 张照片。
4. 期望：选择器正确打开，选择后文件名/缩略图显示，提交成功。

### M-04 上传视频

1. 进入上传页。
2. 选择媒体上传入口。
3. 从系统相册选择 1 个视频。
4. 期望：选择器正确打开，文件信息显示，提交成功。

### M-05 上传头像

1. 进入人物详情页。
2. 点击更换头像。
3. 选择照片。
4. 期望：头像成功更新，页面显示新头像。

### M-06 上传纪事附件

1. 进入上传页 / 纪事编辑页。
2. 选择纪事附件上传入口。
3. 选择图片或视频。
4. 期望：附件成功关联到纪事。

### M-07 访问媒体详情

1. 进入媒体列表。
2. 点击某条媒体。
3. 期望：详情页打开，缩略图/视频可预览。

### M-08 受保护媒体显示与刷新

1. 访问一条需要登录才能查看的媒体。
2. 确认正常显示。
3. 退出登录后再次访问同一链接。
4. 期望：显示未授权/登录提示，不显示过期缓存内容。

### M-09 页面刷新后会话恢复

1. 登录后刷新页面。
2. 期望：保持登录态，不需要重新输入凭证。

### M-10 PWA 安装后重新打开

1. 通过浏览器将应用添加到主屏幕 / 安装 PWA。
2. 关闭应用。
3. 从桌面图标重新打开。
4. 期望：以独立窗口打开，保持在登录态，功能正常。

## 6. 通过标准

- **P0 用例**：所有 `✅` 单元格至少通过一次。
- **自动化回归**：全部通过，且新增平台相关单元测试覆盖率 ≥ 80%。
- **阻塞项**：任一 P0 用例在任一平台失败，禁止全量上线。
